Migration Guidelines
  • Live migration keeps the VM running during the transfer with minimal downtime
  • Offline migration requires the VM to be stopped first for complete data transfer
  • Both nodes must have access to the same shared storage for successful migration
  • Target node must have sufficient CPU, memory, and storage resources available
  • Migration is performed asynchronously in the background and can be monitored
